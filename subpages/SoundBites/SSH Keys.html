SSH Key

SSH, also known as Secure Shell, is a type of network protocol that allows a remote connection between systems, usually used in Linux. SSH is an encrypted protocol and due to the encrypted connection between systems, a key is needed in order to activate the SSH protocol. Keys come in many different bit lengths that correspond to different digit lengths. For example, a 2048-bit encryption is 617 digits long. The larger the number of bits, the longer the code, and the more secure the connection is. Each pair of keys is made of a private and public key. A connection cannot be established unless the local computer contains a key pair, and the server being connected to has a copy of the public key. When creating a key pair, a username is also created. This username and the IP of the remote device are entered into the terminal of the local device in order to establish a remote connection. Using SSH Key management tools keep SSH keys organized, secure, and easy for large companies to manage When a connection is attempting to be established, a random string is generated by the server and is encrypted using the public key. Messages can only be decrypted by the private key, and messages are only sent to devices that contain the private key.
