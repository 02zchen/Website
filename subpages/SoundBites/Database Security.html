Database Security

Databases such as MySQL can be very hard to secure, as they can take input directly from the user, but the risk can be mitigated by securing the database. Some methods of securing a database can be as easy as whitelisting or blacklisting certain symbols and numbers, while also being as hard as setting lowest possible permissions usable to the user, which is not easy in MySQL. An unsecured database can be made to send a list of passwords into a throw away email address, or even crash your server. This is something that maybe your everyday computer user might never encounter, but it always won't harm to be informed about it.
