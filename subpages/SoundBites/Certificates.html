Certificates

Certificates and digital signing are used in the computer world to help validate programs you download have not been tampered with. If the computer says that no certificate could be found, chances are someone has replaced the actual program with malicious software before you even downloaded it. If this is the case, you should not run the program, as it is probably malware. Certificates are used in almost all software updates, to help verify the integrity of the files. Even operating systems like Windows, macOS, and Linux use certificates to check the legitimacy of updates. There are two parties here, the user, you, and the software developers, Microsoft. Each party has a particular key that only works with each other, which is called “asymmetrical encryption”. You have the decryption key, which you downloaded with the rest of the software, and Microsoft has the encryption key, which they encode their data in to keep it secure. Once Microsoft encrypts the data, it is then attached to the original product, which is called a digital signature. Once your computer decrypts it, it can see that the message was encrypted by Microsoft, verifying the integrity of the files.
